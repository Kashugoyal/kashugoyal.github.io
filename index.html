Sawyer Force Control

Description

For any robot the physical properties of any grasped object are the same. Aimed at giving the robot this sense of touch i.e basically allow it to differentiate between a brick and a sponge, this project utilizes ATI force/torque sensor coupled with Baxter's end effector.

The data from the sensor is used by a ROS package that makes the robot 'touch' smart.

Skills Involved

  • ROS
  • Python
  • Inverse Kinematics

Mathematical Simulation of 3 DOF Trebuchet

Description

A trebuchet has three degrees of freedom, one of which is initially constrained as the 'to be projected' mass, rests on the floor. As the counterweight descends, the contact is lost and the mass is launched when the two links align.

After the projectile is launched, the mass impacts a horizontal line at a height of 2.5m from the base. To show realistic results, a coefficient of restitution with a value of 0.5 is added to the impact update law. The angular velocity of mass just changes direction owing to friction.

The entire simulation was done using Mathematica and the detailed report on fproblem formulation and simulation can be found here.

Skills Involved

  • Mathematica
  • Lagrangian Dynamics
  • Rigid Body Simulations

Robotic Manipulation

Description

Robots have the capability to follow precise trajectories at desired velocities and accelerations. Simply put, a robot can move the way you tell you to and at the speed you tell you to.

This project is the application of concepts of Forward and Inverse Kinematics to manipulate the end effector of KUKA youBot along a given time-scaled reference trajectory. A PI feedback controller is used for error compensation. The animation of the simulation is below:

StarBax

Description

Robots are intended to help humans and take over responsibility of repetitive mundane things. Working along these lines wouldn't it be cool if Baxter made coffee for us?

This project is aimed at developing a ROS package to instruct Baxter to amke coffee. Baxter uses its arm vision camera to dtect the objects such as the Keurig, K-cup, and the coffee cup. It then follows instructions in the package to serve hot coffee.

To read more about the project, refer the Readme of the Git repository.

Mecanum Wheel Platform

Description

Mecanum Wheels are the harbingers of omnidirectional automobiles of the future. Though they have become ubiquitous now, they weren't as popular a few years ago and even less popular in India. To learn about omni directional manipulation and develop an industrial scale mecanum wheel platfrom that was capable of carrying loads upto 2 tons, I along with my team built this project.

Being an end to end project, it involved initial CAD design followed by material selection, prototyping which involves minhouse manufacturing and then testing. The project was very elaborate and the learnings were tremendous.

The detailed report on teh project can be seen here.

The CAD model and renderings can be viewed using this link. All modelling and analysis was done using SolidWorks.

Solar Powered Sterling Engine

Description

Energy from renewable sources is coveted but expensive. Solar cells though highly efficient, involve tremendous amount of capital investment. This makes it difficult for economically weak governments to invest in the same. As a cos effective solution to this problem we developed a proof of concept of a sterling engine whose heat source is sun.

We collected sun rays using a cocave mirror which was developed using a mosaic of mirrored glass squares pasted on a concave satelite reciever. The focused heat was used to power the engine.

The detailed report on the project can be seen here.

Skills Involved

  • Prototyping
